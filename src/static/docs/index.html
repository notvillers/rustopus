THE API
THAT SPEAKS
OCTOPUS

HUNGARIAN SOAP/XML IN — CLEAN ENGLISH XML (OR CSV) OUT.

EXPLORE THE API → OPENAPI.YAML

CALL VIA TERMINAL

curl "http://localhost:1140/get-product?url=<octopus-wsdl>&authcode=<code>"

#1 TRANSLATE

HUNGARIAN IN, ENGLISH OUT

FETCHES OCTOPUS 8 ERP SOAP PAYLOADS AND RETURNS THEM AS ENGLISH-TAGGED XML. PASS language=hu TO KEEP THE ORIGINAL HUNGARIAN.

#2 CONVERT

XML OR CSV

EVERY SUPPORTED ENDPOINT CAN ALSO ANSWER IN SEMICOLON-DELIMITED CSV — JUST ADD data_type=csv.

#3 AGGREGATE

ONE CALL, EVERYTHING

/get-bulk COMPOSES PRODUCTS, PRICES, STOCKS, IMAGES AND BARCODES INTO A SINGLE RESPONSE WITH PER-CALL FALLBACK ERRORS.

#4 FORWARD

ORDERS GO BACK (BETA)

/post-order ACCEPTS ENGLISH-TAGGED INPUT AND FORWARDS IT TO OCTOPUS AS HUNGARIAN.

ENDPOINTS

/get-product · /get-stock · /get-price · /get-image · /get-barcode · /get-bulk · /get-invoice · /get-mat · /post-order
